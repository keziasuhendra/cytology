Multimedia Encryption (Video Encryption Algorithm)


Previous Next


Nowadays, the safety of multimedia data can be considered as a very important thing in business. For example, video conferencing application only can be accessed by certain people that are involved in the conference. The same goes to video on demand, this kind of video only can be watched by people who pay for a certain amount to have an access for the video. To keep the data safe, we need to encrypt the multimedia. Multimedia encryption is the core enabling technology that provides confidentiality and prevents unauthorized access of the content. Real time constraints, large amount, and unique characteristics of multimedia data inhibits the use of traditional cryptographic algorithms over multimedia data.

There are several aspects that should be remembered when you want to encrypt a multimedia data. First of all, the encryption should be done real time and the size of the data is quite big. We should choose the right way to encrypt the multimedia data especially video because it can affect the user's experience in watching the video. The wrong move in applying the algorithm to encrypt the video can make the buffering more longer than usual and it will cause an inconvenience towards the user. Many people apply video encyption algorithm to encrypt their videos because it is compatible to be applied to video streaming. This algorithm is based on stream cipher or block cipher depends on the video's needs. The global schema of the algorithm can be defined as below.

  1. Open the MPEG file.
  2. Read the MPEG file and detect the frame type.
  3. Read the stream bit from the frame.
  4. The stream bit will be written directly to the destination file, only if the frame from the stream bit isn't frame type I.
  5. If the stream bit is from frame I, then the system will perform logical exclusive disjunction of the bits.
  6. Write the encryption result to the destination file.
  7. Read the next frame and repeat the process from step 2 until the end of the file.

Video encryption algorithm's secret key, k, will be generated randomly in bitstream with m length, and can be written as k = b1b2...bm. Meanwhile, the bitstream can be represented as S = ..s1...s2...sm...sm+1...sm+2...s2m...  (1). Si (i = 1,2,...) represents all of the bits. The encryption function itself can be represented as Ek(S) = ...(b1 ⊕ s1)...(bm ⊕ sm+1)...(bm ⊕ s2m). In this case, ⊕ represents logical exclusive disjunction operator.
