• Cryptography


In the video encryption algorithm, the logical exclusive disjunction will be performed between the bits in the stream cipher with the bits in the key. Video encryption algorithm can be modified depends on the type of needs of the video. The packages of data will be encrypted using the cipher block DES (Data Encryption Standard). The key function for DES is built by using the MD5 hash function. The modified video encryption algorithm can be defined as below.

  1. The data flows for stream cipher aren't represented with bits anymore but with packages of data which are the results of the output from RTP Socket Adapter.
  2. Each of the data package is encrypted using the secret key DES cryptography algorithm.
  3. Before the key for encrypting the data packages is used by DES, it should be hashed first using the MD5 hash function.
Below is the global schema of the modified video encryption algorithm.
  1. Take one of the data packages from the video file which are the output from RTP Socket Adapter.
  2. The data package is encrypted using the DES cryptography algorithm. The input key for the DES is from the key which is resulted from the input key from the user's input that had been hashed before using MD5 hash function before.
  3. The data package which had been encrypted is sent to the receiver.
  4. Read the next data package and repeat the step from the first one.

There are several video's formats that can be supported such as MPEG-1, Cinepak, H.261, H.263, Indeo32, Indeo41, MJPEGA (Motion JPEG-A), MJPEGB (Motion JPEG-B), and MJPEG (Motion JPEG). The function of media locator is to find the location of the string that had been given by the user. The processor is to read the file from media locator and make sure that the file is ready to be sent by send stream. The data source is the representation of the file that is ready to be sent by the send stream that has the format of RTP compliant.

The RTP manager will initialize the connection and create the send stream object. Meanwhile, send stream will read the data source object and send all of the film's frames, including to organize the buffer usage for sending all of the frames. RTP Socket Adapter will take each of the frames from data source which is going to be sent to the encryptor to be decrypted. The encryptor's task is to encrypt all of the frames using the algorithm and the keyword that had been given. The encryptor is made using DES and the keyword will be changed using MD5 into a new keyword with the same length. The hashed key will be used as the key in DES algorithm. The encryptor will save the result of encryption in output stream buffer which will send all of the datas that are saved in the buffer.

Now, this paragraph will discuss about the receiver. RTP manager is to create the receive stream. The receive stream will receive the RTP stream, then it is going to decrypt the data and change the RTP format into video format that can be played in the player. The input stream buffer receives the RTP stream, organizes the buffer usage for the receiving process, and sends the received data to the decryptor which will decrypt the data using the given key. Meanwhile, RTP socket adaptor changes the RTP format into the suitable format that can be read by the player. The player will receive the film format that can be played and create the controller to display the interface for user to set the events. Frame that is received by the player will be parsed and sent to the direct draw codec to be displayed in the monitor.
